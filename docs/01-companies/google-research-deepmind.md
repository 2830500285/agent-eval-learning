# Google Research and DeepMind / Google Research 与 DeepMind

## English

Google's evaluation output spans academic benchmarks, agent system studies, and domain-specific user studies.

### Key resources

1. Towards a science of scaling agent systems: When and why agent systems work
   - Link: https://research.google/blog/towards-a-science-of-scaling-agent-systems-when-and-why-agent-systems-work/
   - Why it matters: A controlled study of many agent configurations across model families, useful for understanding when multi-agent setups help or hurt.

2. A scalable framework for evaluating health language models
   - Link: https://research.google/blog/a-scalable-framework-for-evaluating-health-language-models/
   - Why it matters: Good example of combining human and automated evaluation in a high-stakes domain.

3. Towards better health conversations: Research insights on a wayfinding AI agent
   - Link: https://research.google/blog/towards-better-health-conversations-research-insights-on-a-wayfinding-ai-agent-based-on-gemini/
   - Why it matters: Shows user-study-driven evaluation of an applied agent, not just static benchmark scores.

4. Revisiting the Arcade Learning Environment: Evaluation Protocols and Open Problems for General Agents
   - Link: https://research.google/pubs/revisiting-the-arcade-learning-environment-evaluation-protocols-and-open-problems-for-general-agents/
   - Why it matters: Older but still useful for understanding protocol discipline in general-agent evaluation.

### Google pattern

Google and DeepMind material is often valuable when you want more experimental rigor around agent architectures, evaluation protocols, and user-study design.

### How to use Google material in your own stack

1. Use Google Research and DeepMind material when you need stronger experimental discipline.
2. Compare one variable at a time when testing agent architectures.
3. For high-stakes domains, combine automated metrics with expert review and user studies.
4. Treat protocol design as part of the evaluation problem.

## 中文

Google Research / DeepMind 的 eval 内容横跨学术 benchmark、agent system 研究以及领域化用户研究。

### 关键资料

1. Towards a science of scaling agent systems
   - 链接：https://research.google/blog/towards-a-science-of-scaling-agent-systems-when-and-why-agent-systems-work/
   - 价值：系统比较多种 Agent 架构在不同模型家族上的表现，适合理解多 Agent 什么时候有效、什么时候反而拖后腿。

2. A scalable framework for evaluating health language models
   - 链接：https://research.google/blog/a-scalable-framework-for-evaluating-health-language-models/
   - 价值：高风险领域中，人评和自动评结合的一个典型例子。

3. Towards better health conversations
   - 链接：https://research.google/blog/towards-better-health-conversations-research-insights-on-a-wayfinding-ai-agent-based-on-gemini/
   - 价值：展示了如何通过用户研究来评估应用型 Agent，而不是只看静态 benchmark。

4. Revisiting the Arcade Learning Environment
   - 链接：https://research.google/pubs/revisiting-the-arcade-learning-environment-evaluation-protocols-and-open-problems-for-general-agents/
   - 价值：虽然较早，但对“评测协议本身必须严谨”这个问题仍然很有启发。

### Google 的风格

如果你关心 Agent 架构实验设计、评测协议严谨性、以及用户研究方法，Google 系的材料通常参考价值很高。

### 如何把 Google 的思路用到自己的系统里

1. 当你希望实验设计更严谨时，Google 系材料特别值得参考。
2. 测试 Agent 架构时，尽量做到一次只改一个变量。
3. 在高风险领域里，自动指标、专家审核和用户研究要结合。
4. 把“评测协议如何设计”本身也当成评测问题的一部分。
